Для создания справочных материалов используется облегченный язык разметки **Markdown**.

Он позволяет форматировать электронные документы так, чтобы их исходный текст оставался максимально читаемым для человека.

Все файлы, размеченные в этом формате, должны иметь расширение **.md** и размещаться внутри определенных папок.

Каждая такая папка фактически является отдельным разделом справочной системы, участвуя в динамическом формировании оглавления на основе текущей иерархической и внутреннего содержимого папок.

При изменении иерархии или содержимого папок оглавление справочных материалов будет перестраиваться автоматически.

Внутри каждой папки могут располагаться следующие элементы:

1. Файлы в формате **Markdown** со справочными материалами.
1. Вложенные папки, которые будут рассматриваться как подразделы данного раздела справочной системы.

Все эти элементы должны быть описаны в специальном файле с именем **_.dir**, который должен находится внутри каждой папки.

Все элементов справочной системы описываются в этом файле с помощью формата JSON следующим образом:

```json
[
    {
        "name": "Имя файла или вложенной папки",
        "label": "<b style='color:blue'>Название пункта</b> оглавления",
        "content": {
            "abstract": "\n~~~md\n Краткая аннотация.\n~~~"
        }
    },
]
```

Данный формат подразумевает, что каждый элемент папки, участвующий в формировании оглавления, должен задаваться отдельной записью внутри общего JSON-массива.

Каждая запись может содержать несколько ключей, два из которых являются обязательными:

1. Ключ **name** — определяет либо имя файла, в котором хранится текст со справочными материалами, либо имя вложенной папки, содержимое которой должно стать подразделом внутри данного раздела справочной системы.

1. Ключ **label** – задает значение, которое будет выводиться в качестве названия данного пункта оглавления.

Необязательный ключ **content** позволяет задать дополнительные настройки отображения каждого пункта. Например, с помощью ключа **abstract** можно задать краткое описание (аннотацию) у данного пункта оглавления. Текст этого описания можно дополнительно отформатировать в формате **Markdown**, если записать его между специально предусмотренных тегов **~~~md** и **~~~**.

Если текст оповещения требуется дополнительно отформатировать в стандарте «**Markdown**», то в заголовок необходимо добавить специальный модификатор **md** через символ подчеркивания (_).

Например,

```text
    ```info_md_
        **Markdown** внутри `markdown`
    ```
```

```info_md
    **Markdown** внутри `markdown`
```

```info _md
    **Markdown** внутри `markdown`
```

```info _md_
    **Markdown** внутри `markdown`
```

```info_md_
    **Markdown** внутри `markdown`
```

Без этого модификатора любые элементы разметки «**Markdown**» в тексте оповещения будут восприниматься в буквальном смысле, т.е. они будет выводится как обычные символы без форматирования текста.

Например,

```text
    ```info
        **Markdown** внутри `markdown`
    ```
```

```info
    **Markdown** внутри `markdown`
```

Кроме модификатора «**md**» стандартный синтаксис подсветки кода был расширен дополнительными модификаторами. Они указываются после названия языка программирования через пробел.

Например,

```text
    ```javascript дополнительные_теги
    ```
```

Каждый дополнительный модификатор должен отделяться от другого знаком подчеркивания.

Эти модификаторы имеют следующее назначение:

1. Модификатор **run** говорит о том, что текст кода будет не только по определенному подсвечиваться, но и выполняться.

```text
    ```javascript run
        let newElement = document.createElement('div');
        newElement.innerHTML = '<input placeholder="Модификатор run">';
        document.body.append(newElement);
    ```
```

При его указании под редактором появится дополнительная область, в которой будет выводится результат выполнения самого кода.

Например,

```javascript run_[my-component.js]
    let newElement = document.createElement('div');
    newElement.innerHTML = '<input placeholder="Модификатор run">';
    document.body.append(newElement);
```

1. Модификатор **edit** говорит о том, что текст кода можно будет редактировать.

```text
    ```javascript run_edit
        let newElement = document.createElement('div');
        newElement.innerHTML = '<h1>Модификатор edit</h1>';
        document.body.append(newElement);
    ```
```

Например,

```javascript run_edit_[my-component.js]
    let newElement = document.createElement('div');
    newElement.innerHTML = '<input placeholder="Модификатор run">';
    document.body.append(newElement);
```

В этом случае 

1. Модификатор ODA-компонента **[]** говорит, что указанный код будет подключен в виде скрипта к области вывода результата выполнения кода. Это имя состоит из частей:

1. Имени компонента, например, my-component.
1. Расширения **js**.

Например,

```text
    ```javascript run_edit_[my-component.js]
    ODA({
        is: 'my-component',
        template: `
            <span> Модификатор h</span>
        `,
    });
    ```
```

Кроме этого, в область вывода автоматически добавляется HTML-тег по имени данного компонента без расширения **js**, который станет хостом данного компонента.

```javascript run_edit_[my-component.js]
    ODA({
        is: 'my-component',
        template: `
            <span> ODA-компонент</span>
        `,
    });
```

В результате этого в области вывода будет выводится содержимого самого компонента, код которого записан в редакторе кода.

При указании этого модификатора под самим кодом появится дополнительная область, в которой будет выводится результат его выполнения.

```javascript run_[my-component.js]
    let newElement = document.createElement('div');
    newElement.innerHTML = '<input placeholder="Модификатор run">';
    document.body.append(newElement);
```

Любое изменение текста кода в этом случае будет приводить к его автоматическому выполнению.

```javascript run_edit_[my-component.js]
    let newElement = document.createElement('div');
    newElement.innerHTML = '<h1>Модификатор edit</h1>';
    document.body.append(newElement);
```

В этом примере можно изменить текст заголовка в самом коде и убедиться, что результат будет изменяться динамически при вводе любого символа.

1. Модификатор **line** позволяет добавить автоматическую нумерацию строк кода.

```text
    ```javascript run_edit_line_[my-component.js]
        let newElement = document.createElement('div');
        newElement.innerHTML = '<h1>Модификатор line</h1>';
        document.body.append(newElement);
    ```
```

В этом случае с правой стороны от источника кода появляется дополнительная полоса, на которой последовательно выводятся номера строк.

```javascript run_edit_line_[my-component.js]
    let newElement = document.createElement('div');
    newElement.innerHTML = '<h1>Модификатор line</h1>';
    document.body.append(newElement);
```

1. Модификатор **console** позволяет отобразить дополнительную область для вывода сообщений, отправленных в консоль.

```text
    ```javascript run_edit_line_console_[my-component.js]
        for(var i = 0; i < 5; ++i)
            console.log("Hello, console!");
    ```
```

При его использовании в правом верхнем углу окна с кодом появится кнопка, при нажатии на которую развернется дополнительная область со всеми сообщениями консоли.

```javascript run_edit_line_console_[my-component.js]
    for(var i = 0; i < 5; ++i) {
        console.log("Привет, консоль!");
    }
```

В правом верхнем углу этого окна консоли располагаются три кнопки управления:

1. Кнопка «**Clear**» — очищает область консоли, удаляя все сообщения из нее.

1. Кнопка «**Refresh**» — выполняет код повторно, предварительно удаляя все предыдущие сообщения.

1. Кнопка «**Close**» — закрывает область консоли.

1. Модификатор **loadoda** позволяет автоматически подключить библиотеку «**oda.js**».

```text
    ```javascript run_edit_line_console_loadoda_[my-component.js]
        ODA({
            is: 'my-component',
            template: `
                <span> Модификатор loadoda</span>
            `,
        });
    ```
```

При его использовании не нужно вручную подключать библиотеку будет работать в правом верхнем углу окна с кодом появится кнопка, при нажатии на которую развернется дополнительная область со всеми сообщениями консоли.

```javascript run_edit_line_console_[my-component.js]
    ODA({
        is: 'my-component',
        template: `
            <span> Модификатор loadoda</span>
        `,
    });

Без его использования библиотеку «**oda.js**» придется подключать с помощью тега «**script**» следующим образом:

Например,

```text

1. Модификатор **eh** позволяет задать высоту области редактора кода.

```text
    ```javascript run_edit_line_console_loadoda_[my-component.js]_h=180_

Высота области редактора кода указывается в пикселях. Эти единицы измерения отдельно не указываются. После значения высоты обязательно записать один символ подчеркивания.

Для этого сначала необходимо задать родительские компоненты в любом месте страницы.

```text
    ```javascript _run_line_edit_console_[base-component-1.js]
    ODA({
        is: 'base-component-1',
        template: `
            <div>Родитель 1</div>
        `
    });

    ```javascript _run_line_edit_console_[base-component-2.js]
    ODA({
        is: 'base-component-2',
        template: `
            <div>Родитель 2</div>
        `
    });
    ```
```

После этого необходимо в фигурных скобках перечислить имена всех компонентов, которые планируются для использования в качестве родителей данного компонента. Только после этого наследник сможет указать их в своем списке **extends** через запятую.

```text
    ```javascript _run_line_edit_[my-component.js]_{base-component-1.js_base-component-2.js}
        ODA({
            is: 'my-component',
            extends: 'base-component-1, base-component-2',
            template: `
                <div>Наследник</div>
            `
        });
    ```
```

ODA({
    is: 'base-component-1',
    template: `
        <div>Родитель 1</div>
    `
});
```

Задание второго родителя

```javascript _run_line_edit_console_[base-component-2.js]

`
});
```

```javascript _run_line_edit_[my-component.js]_{base-component-1.js_base-component-2.js}

    is: 'my-component',
    extends: 'base-component-1, base-component-1',
    template: `

    `

В этом примере с помощью механизма CDN и технологии JavaScript модулей сначала осуществляется подключение библиотеки фреймворка. Для этого обязательным условием является использование в теге **script** атрибута **type** со значением **module**.

В теле HTML-страницы задается пользовательский тег <my-component>, который, однако, не будет отображаться браузером по умолчанию, так как он не является его нативным HTML-элементом.

Для отображения этого тега в браузере необходимо с помощью функции ODA нашего фреймворка зарегистрировать пользовательский компонент с таким же именем.

Давайте подробнее разберем код самого компонента, который передается этой функции в списке параметров в виде объекта с определенными свойствами:

1. Свойство **is** этого объекта определяет имя будущего компонента, под которым он будет зарегистрирован в браузере.

1. Свойство **template** задает шаблон будущего компонента, при написании которого используется формат JSX.

1. Свойство **props** содержит объявления всех свойств компонента, которые будут использоваться для хранения его данных. В данном примере это свойство **text**, которое будет автоматически подставлено в содержимое элемента **span** с возможностью динамического изменения его значения.

На основе этой информации функция ODA сама создаст и зарегистрирует пользовательский компонент, а браузер получит возможность отображать его, используя кастомный тег компонента как нативный HTML-элемент.
